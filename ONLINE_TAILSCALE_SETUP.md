# Онлайн-подключение через Tailscale

Этот проект уже умеет работать в двух режимах:
- `Локальная база`
- `Онлайн база`

Для текущей машины серверный адрес Tailscale:

```text
100.86.4.84
```

Именно этот адрес нужно использовать в клиентском приложении как `host`.

## Что уже подготовлено в проекте

- В `config.ini` для профиля `database_online` уже подставлен `100.86.4.84`.
- Активный профиль тоже сейчас указывает на `100.86.4.84`.
- В приложении уже есть выбор между локальной и онлайн-базой.
- Для клиента создан отдельный пользователь базы `cost_client_app`.

## Что настроено на серверном компьютере

### PostgreSQL уже слушает сеть

Проверенные параметры `postgresql.conf`:

```conf
listen_addresses = '*'
port = 5432
```

### Доступ из Tailscale уже разрешен

В `pg_hba.conf` добавлено правило:

```conf
host    all    all    100.64.0.0/10    scram-sha-256
```

### Служба PostgreSQL уже перезапущена

После добавления правила служба была перезапущена, и проверка подключения по адресу `100.86.4.84` прошла успешно.

## Актуальные параметры подключения для клиента

```ini
[database_online]
host = 100.86.4.84
port = 5432
name = cost_online_demo
user = cost_client_app
password = CostClientApp_2026!
sslmode = disable
sslrootcert =
```

## Почему используется отдельный пользователь

Для клиентского приложения лучше использовать не `postgres`, а отдельного пользователя БД.

Сейчас рабочим пользователем приложения является:

```text
cost_client_app
```

Ему передано владение рабочими объектами онлайн-базы, потому что приложение местами выполняет `ALTER TABLE IF NOT EXISTS` во время работы.

## Что важно перед началом работы

- Серверный компьютер должен быть включен.
- Серверный компьютер не должен уходить в сон.
- Tailscale на сервере должен быть в статусе `Connected`.
- Клиентский компьютер тоже должен быть подключен к тому же tailnet.
- Для клиента должен использоваться адрес `100.86.4.84`, а не `localhost`.

## Рекомендации по безопасности

- Не используйте пользователя `postgres` в клиентском приложении.
- Не открывайте порт PostgreSQL напрямую в интернет, если работает Tailscale.
- Делайте регулярный дамп базы.
- Если будете переносить базу новым дампом, следите, чтобы владельцем рабочих таблиц оставался пользователь приложения.
