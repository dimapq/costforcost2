# Чек-лист проверки подключения с другого компьютера

Этот список помогает быстро понять, где именно проблема, если клиент не может войти в онлайн-базу.

## Быстрый чек-лист

1. На серверном компьютере Tailscale показывает `Connected`.
2. На серверном компьютере Tailscale IP равен `100.86.4.84`.
3. На клиентском компьютере Tailscale тоже показывает `Connected`.
4. Оба компьютера находятся в одном tailnet.
5. На серверном компьютере работает служба `postgresql-x64-17`.
6. В `postgresql.conf` указано:

```conf
listen_addresses = '*'
port = 5432
```

7. В `pg_hba.conf` есть строка:

```conf
host    all    all    100.64.0.0/10    scram-sha-256
```

8. В приложении выбран режим `Онлайн база`.
9. В приложении указан адрес `100.86.4.84`.
10. В приложении используются актуальные учетные данные:

```ini
user = cost_client_app
password = CostClientApp_2026!
```

## Если не работает

### Симптом: база не находится

Проверьте:
- включен ли серверный компьютер;
- подключен ли Tailscale на обеих машинах;
- не указан ли `localhost` вместо `100.86.4.84`.

### Симптом: ошибка логина или пароля

Проверьте:
- логин `cost_client_app`;
- пароль `CostClientApp_2026!`;
- нет ли лишних пробелов при вставке.

### Симптом: приложение подключается, но некоторые действия падают

Проверьте:
- использует ли приложение именно пользователя `cost_client_app`;
- не переключился ли профиль обратно на локальную базу;
- не была ли онлайн-база заменена другим дампом с чужими владельцами таблиц.

## Контрольная проверка через psql

Если нужно проверить вручную с клиентского компьютера:

```bash
psql -h 100.86.4.84 -p 5432 -U cost_client_app -d cost_online_demo
```

Если вход открылся, значит сетевая часть настроена правильно.
